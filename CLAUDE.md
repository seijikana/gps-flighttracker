## プロジェクト概要

GPSによる自車位置・走行軌跡トラッキングと、RTL-SDR（ADS-B受信）による周辺航空機トラッキングを
**同一の地図上に重ねて表示する**車載Webアプリ。Raspberry Pi 5に搭載し、HDMI接続のカーナビ画面に表示する。

着想元: Cameron Pachecoによる "Skylight" プロジェクト（ADS-B受信機＋プロジェクターで
天井に航空機の動きをリアルタイム投影するDIYフライトトラッカー）。本プロジェクトでは
天井投影ではなく、Webダッシュボード上の地図表示として実現する。

現時点では独立アプリとして実装するが、将来`raspi5-openclaw`配下のcariotダッシュボード
（Flask, port 5000）と並べて閲覧できるよう、ポート・API・静的ページを分離した構成にする
（統合方法・タイミングは未確定、今回は分離設計のみ用意）。

## ハードウェア

| デバイス | 用途 |
|---|---|
| NESDR Mini 2+ (0.5PPM TCXO) RTL-SDR | 1090MHz ADS-B信号受信（航空機の位置・高度・速度等） |
| DOCTORADIO GR7-10HZ | 自車位置・走行ルート用GPS（USB接続、アンテナ内蔵、10Hz更新） |
| GY-521 (MPU-6050) IMU | 走行セッション判定（停車/振動検知）に利用、`raspi5-openclaw`側で接続済み |
| MAX98357A I2Sアンプ | 新規機体検出時の通知音再生に流用（音声アシスタントモジュールと共用） |
| ZTEルーター (povo2.0 SIM) | 車載WiFi。電源はiot-pvcontrol側で管理（Raspi5と同様） |
| USB HDD | オフライン地図タイル（日本全国分）の保存先 |

ADS-B受信用アンテナはNESDR Mini 2+付属の1090MHz専用アンテナを使用する想定
（車のシャークフィン型AM/FM/GPS統合アンテナ等の流用は帯域が合わず不可）。

### 車載WiFi（povo2.0）について

povo2.0 SIMを挿したZTEルーターを車載運用する。無料枠は1228kbpsで帯域制限があり、
外部API（FR24/OpenSky等）への問い合わせやタイル取得で帯域が不足する場合は
「データ追加（トッピング）60GB/180日」を購入する運用とする。トンネル等での回線断も
発生する前提のため、**オフラインファースト**（自車位置・受信中航空機の表示は通信なしでも継続）
で設計する。

## 要件サマリー

### 1. 自車位置・走行軌跡（GPS + IMU）

- gpsd経由でGPS（GR7-10HZ）から位置・速度を取得し、SQLiteに蓄積する。
- GPS速度・IMU振動から「走行セッション」を判定し、現在進行中のセッションとして逐次記録する
  （`raspi5-openclaw`/iot-pvcontrolの走行/停車判定ロジックを参考にする）。
- **セッション分断の閾値**: 信号待ち・渋滞等の短時間停車でセッションが切れないよう、
  「停車1時間以上 **かつ** IMU振動なし」を満たした場合のみセッション終了とする。
  短時間の停車・アイドリングはセッションを継続する。
- 過去の走行セッションもすべて記録し、UIから一覧表示・呼び出して地図上に再表示できるようにする。

### 2. 航空機トラッキング（ADS-B）

- dump1090(-fa) または readsbでADS-B信号を受信・デコードし、JSON出力（aircraft.json等）を取得する。
- **現在検出中の機体のみ**を軌跡として地図に重畳表示する（ロストした機体の軌跡は表示を止める）。
- **新規機体検出時にRaspi本体のスピーカーから通知音を発信する**
  （音声アシスタントモジュールのMAX98357A/I2Sアンプを流用し、簡易な検出音再生処理を実装する）。

### 3. 航空機情報の補完

dump1090のJSON出力（icao, lat/lon, alt, speed, track, flight等）だけでは
エアライン・出発地/目的地・機種情報が不足するため、外部APIで補完する：

- **機種**: ICAO24をキーに、機体データベース（OpenSky `aircraftDatabase.csv`等）をローカルキャッシュして参照。
- **エアライン・出発地/目的地**: FlightRadar24（非公式API）またはOpenSky Network REST APIから
  コールサイン/ICAO24をキーに取得する。レート制限・利用規約を確認した上で実装すること。
- **オフライン時のフォールバック**: 通信不可時は補完処理をスキップし、dump1090から得られる
  範囲の情報（位置・高度・速度・コールサイン）のみで表示を継続する。

### 4. FlightRadar24フィーダ参加

ADS-B受信データをFR24にフィードし、Business/Enterpriseプラン相当のデータアクセス権を
無償取得する。以下の手順をセットアップ作業として実装・整備する：

1. `fr24feed`パッケージをインストールする。
2. `fr24feed-config.sh`で受信機シリアル・緯度経度等を設定する。
3. dump1090の出力をfr24feedに供給する（Beastフォーマット転送、ポート30005等）。
4. FR24マイページでフィーダを登録し、データ品質に応じた無償プラン付与条件を確認する。

### 5. 地図表示（オフライン対応・自動追従）

- 地図描画はLeaflet.js + OpenStreetMap（OpenMap）タイルを使用。
- **日本全国分のタイルをUSB HDDに事前ダウンロード**し、ローカルタイルとして参照する
  （タイルダウンロードツールの選定、USB HDDへの配置、Leaflet側でのローカルタイルパス設定を行う）。
- 自車位置: GPSから現在地マーカー＋走行中セッションの軌跡ポリラインを描画。過去セッションは
  UIから選択して呼び出し表示できるようにする。
- 航空機: dump1090のJSON出力をポーリングし、現在検出中の機体のみをアイコン＋
  コールサイン/高度/速度/エアライン/機種等のラベル付きマーカーとして重畳表示、一定間隔で位置更新する。
- **HDMI接続のカーナビ画面はタッチパネル操作不可**（手動でのズーム/パン操作ができない）のため、
  自車の走行軌跡（現在位置・進行方向）に追従して**自動でズーム・パンを調整**するUIロジックを実装する。

## アーキテクチャ方針

- 独立したFlask/FastAPIサービスとして新規ポート（例: 5001）で起動し、cariot(port 5000)とは
  プロセス・ポートを分離する。
- 将来cariotダッシュボードに「iframe埋め込み」または「リンクページ追加」で統合できるよう、
  バックエンドAPIとフロントエンド（静的ページ）を分離した構成にする。
- データ層: SQLite（走行セッション軌跡ログ、航空機トラック履歴）。cariotの`history_store.py`の
  設計パターン（ロールアップ等）は参考にしてよいが、DBファイルは新規・独立とする。

## 想定ディレクトリ構成（実装時に確定）

```
gps-tracker/
  app.py / api.py        - Flask/FastAPIサービス（port 5001想定）
  gps_session.py          - GPS+IMUによる走行セッション判定・記録
  adsb_tracker.py          - dump1090/readsb出力のポーリング・現在検出機体の管理
  aircraft_enrich.py       - ICAO24/コールサインからの機種・エアライン・出発地目的地補完
  notify_sound.py          - 新規機体検出時の通知音再生（MAX98357A経由）
  static/                  - Leaflet地図フロントエンド
  tiles/ または USB HDDマウントパス - 日本全国オフラインタイル
  tracker.db               - 走行セッション・航空機トラックのSQLite DB
```
