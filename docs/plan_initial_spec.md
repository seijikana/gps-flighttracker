# gps-tracker webapp 構築プロンプト 作成計画

## Context

`gps-tracker/CLAUDE.md` には、Raspi5 + GPS（DOCTORADIO GR7-10HZ）+ ADS-Bレシーバー（NESDR Mini 2+ / dump1090）を使い、自車軌跡と周辺航空機をオフライン地図上に重ねて表示するアプリの構想が記載されているが、コードは未着手。今回は実装そのものではなく、**この実装をClaude Codeに依頼するためのプロンプト（指示文書）を作成する**ことが目的。

ユーザーから以下の方針が確定：
- 独立アプリとして実装するが、将来`raspi5-openclaw`配下のcariotダッシュボード(Flask, port 5000)への統合を見据えたポート分離・モジュール構成にする
- FlightRadar24等への「フィーダ参加」（dump1090出力をfr24feed等に転送し、Business/Enterpriseプラン相当を無償取得する手順）もプロンプトに含める
- オフライン地図タイルは日本国内（自車移動範囲）のみ事前キャッシュする設計

車載WiFi(povo2.0のテザリング)はモバイル回線のため、機内/トンネル等で切れる前提のオフラインファーストな設計が必要。

## 作成するプロンプトの構成

`gps-tracker/CLAUDE.md` を更新・拡充する形で、Claude Codeへの実装依頼プロンプトを作成する。プロンプトには以下を含める：

1. **要件サマリー**
   - GPSで自車位置・走行軌跡を記録（gpsd経由、SQLite蓄積）。GPS+IMU情報から「走行セッション」を判定し、逐次最新セッションとして記録（iot-pvcontrolの走行/停車判定ロジックを流用）。信号待ち・渋滞等での停車でセッションが分断されないよう、**「停車1時間以上 かつ IMU振動なし」を満たした場合のみセッション終了**とする閾値を設ける（短時間の停車・アイドリングはセッション継続）
   - dump1090(-fa)/readsbでADS-B受信、現在検出中の機体のみを軌跡として地図に重畳表示
   - **新規機体検出時にRaspi本体スピーカーから通知音を発信**（音声アシスタント用I2Sアンプ(MAX98357A)を流用、簡易な検出音再生処理を実装）
   - 航空機情報（エアライン/高度/速度/出発地・目的地/機種）はdump1090のJSON単体では不足するため、外部API（FlightRadar24, OpenSky Network, ADSBdb等）からICAO24/コールサインをキーに補完する
   - 地図表示はHDMI接続のカーナビ画面上で操作不可（タッチパネルでの拡大縮小ができない前提）のため、**自車の走行軌跡に追従して自動でズーム・パンを調整**するUIロジックを実装。過去の走行セッションも記録し、UIから一覧・呼び出して表示できるようにする
   - 地図はLeaflet.js + 事前キャッシュした日本全国タイル（オフライン動作前提、povo2.0回線断時も自車・既知機の表示は継続できる設計）

2. **アーキテクチャ方針**
   - 独立Flask/FastAPIサービスとして新規ポート（例: 5001）で起動、cariot(5000)とは分離
   - 将来cariotダッシュボードへ「iframe埋め込み or リンクページ追加」で統合できるよう、API/静的ページを分離した構成にする
   - データ層: SQLite（軌跡ログ・既知機トラックの履歴保存）、cariotのhistory_storeパターンを参考にしてよいが新規DBファイルとする

3. **機体情報補完ロジック**
   - dump1090のaircraft.json（icao, lat/lon, alt, speed, track, flight等）をベースに、コールサイン/ICAO24から下記を補完：
     - 機種: ICAO24ベースの機体データベース（OpenSky `aircraftDatabase.csv`等、ローカルキャッシュ推奨）
     - 出発地/目的地・エアライン: FlightRadar24非公式API or OpenSky Network REST APIから取得（レート制限・利用規約に注意する旨を明記）
   - 通信不可時（オフライン時）は補完をスキップし、dump1090から得られる範囲の情報のみ表示するフォールバックを設計

4. **FlightRadar24フィーダ参加手順**
   - `fr24feed`パッケージのインストール、`fr24feed-config.sh`での設定（受信機シリアル/緯度経度登録）
   - dump1090の出力をfr24feedに供給する設定（Beastフォーマット転送、ポート30005等）
   - フィーダ登録によりFR24 Business/Enterpriseプラン相当の無償アクセス権を得る流れ（マイページでのフィーダ登録、データ品質に応じた付与条件の確認）をプロンプト内に手順として明記

5. **オフライン地図タイル**
   - **OpenStreetMap（OpenMap）の日本全国分タイルをUSB HDDに事前ダウンロード**して運用する手順（タイルダウンロードツール選定、USB HDDへの配置、Leaflet側でのローカルタイルパス参照設定）を明記
   - 自車軌跡ポリライン・航空機マーカーの描画はキャッシュタイル上に重畳
   - カーナビ画面では手動操作不可のため、上記の自動追従ズーム/パンと組み合わせて動作する前提で設計する

6. **未確定事項として明記する点**
   - cariotとの統合タイミング・方法は今後判断（今回は分離設計のみ用意）
   - 補完元API（FR24非公式API vs OpenSky）の最終選定は実装時の利用規約確認次第
   - タイルキャッシュの具体的な地理範囲・ズームレベルは実装時に検討
